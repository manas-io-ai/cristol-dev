1. What is OpenClaw and Why You Want It

OpenClaw is an open-source gateway that connects AI models (like Claude from Anthropic or GPT from OpenAI) to your favorite messaging apps. Think of it as the bridge between a powerful AI brain and the chat apps you already use every day.

‚ùå Without OpenClaw

Open browser ‚Üí go to claude.ai ‚Üí type question ‚Üí get answer ‚Üí close tab. Repeat forever.

‚úÖ With OpenClaw

Text your AI on WhatsApp at 2 AM: "what's on my calendar tomorrow?" ‚Äî it responds like a friend.

Why not just use ChatGPT or Claude directly?

  • Convenience. Your AI lives in your messaging app. No logging in, no switching tabs.
  • Always-on. OpenClaw runs as a background service. It's there 24/7.
  • Multi-channel. Same assistant across WhatsApp, Telegram, Discord, iMessage, Slack, Signal, and more.
  • Customizable. You choose the AI model, configure its personality, give it tools and skills.
  • Private. It runs on YOUR machine. Your conversations stay under your control.
  • Extensible. Skills, plugins, cron jobs, webhooks, browser control ‚Äî it's a platform.

What you can actually do with it

  • Personal assistant: "Remind me to call the dentist at 3 PM" ‚Äî it sets a cron job and pings you.
  • Research helper: "Find me the best restaurants in Lisbon with outdoor seating" ‚Äî web search, curated results.
  • Developer tool: "Check staging and tell me if the deploy went through" ‚Äî it runs the commands.
  • Content creation: "Draft a LinkedIn post about our launch" ‚Äî done in 30 seconds.
  • Smart home brain: Connected to your phone, it can snap photos, get location, read notifications.

2. What You'll Need (Prerequisites)

Hardware: A computer that stays on

OpenClaw runs as a service on your computer. It needs to be on and connected to the internet. Best options:

  • Mac mini ‚Äî ideal. Low power, always on, handles everything.
  • Any Mac or Linux machine ‚Äî MacBook works but needs to stay awake.
  • Linux server or VPS ‚Äî for the technically adventurous.
  • Windows with WSL2 ‚Äî works, but Mac/Linux is strongly recommended.

üí° Tip: A Mac mini is the gold standard for personal OpenClaw setups. Low power, rock solid, macOS makes everything easier.

Software: Node.js 22+

OpenClaw is built on Node.js. You need version 22 or newer. The installer handles this in most cases. To check:

node --version

If you see v22.x.x or higher, you're good.

An AI Provider API Key

OpenClaw doesn't include the AI brain ‚Äî you bring your own. You need an API key from at least one provider:

  • Anthropic (Claude) ‚Äî recommended. Sign up at console.anthropic.com.
  • OpenAI (GPT) ‚Äî alternative. Sign up at platform.openai.com.
  • Others ‚Äî OpenRouter, Groq, local models via Ollama, and more.

üí∞ Cost reality check: API usage costs money. Claude Sonnet runs ~$3‚Äì15/month for moderate use. Heavy Opus usage can be $50‚Äì100+/month. Start with Sonnet.

Optional but recommended

  • Brave Search API key ‚Äî free tier at brave.com/search/api. Without it, no web search.
  • A second phone number ‚Äî for WhatsApp setup (more on this later).
  • Tailscale ‚Äî free secure remote access to your gateway.

3. Step-by-Step Installation

Run the installer

Open Terminal (Mac) or your terminal emulator (Linux):

curl -fsSL https://openclaw.ai/install.sh | bash

On Windows (PowerShell):

iwr -useb https://openclaw.ai/install.ps1 | iex

This downloads OpenClaw, installs it globally, handles Node.js, and launches the onboarding wizard.

The onboarding wizard

The wizard starts automatically. If not, run:

openclaw onboard --install-daemon

It walks you through:

  • Model/Auth ‚Äî pick your AI provider, paste your API key
  • Workspace ‚Äî where agent files live (default is fine)
  • Gateway ‚Äî port and auth mode (defaults are fine)
  • Channels ‚Äî messaging apps to connect (can add later)
  • Daemon ‚Äî install as system service. Say yes for always-on.

Pick QuickStart when asked. It uses sensible defaults.

Verify it's running

openclaw gateway status

Should show the gateway is running. If not: openclaw gateway start

Open the dashboard

openclaw dashboard

Opens http://127.0.0.1:18789/ ‚Äî chat with your assistant in the browser. If this works, your AI is alive. üéâ

Run the doctor

openclaw doctor

Checks your setup for common issues and tells you what to fix.

4. Connecting to Telegram

Telegram is the easiest channel to set up.

Create a bot with BotFather

Open Telegram, search for @BotFather (verified, blue checkmark). Send /newbot, follow the prompts, and save the bot token it gives you.

Add the token to your config

Edit ~/.openclaw/openclaw.json:

{
  channels: {
    telegram: {
      enabled: true,
      botToken: "YOUR_TOKEN_FROM_BOTFATHER",
      dmPolicy: "pairing",
      groups: { "*": { requireMention: true } }
    }
  }
}

Restart and approve yourself

openclaw gateway restart

Message your bot on Telegram. Then approve the pairing:

openclaw pairing list telegram
openclaw pairing approve telegram <CODE>

That's it. Your Telegram bot is live.

Optional: Add to group chats

Add the bot to a group. By default it only responds when @mentioned. To see all messages: BotFather ‚Üí /setprivacy ‚Üí Disable, then remove and re-add the bot to the group.

5. Connecting to WhatsApp

WhatsApp is trickier because it uses WhatsApp Web linking ‚Äî there's no bot API.

‚ö†Ô∏è Use a separate phone number. If you link your personal WhatsApp, every message becomes AI input. Get a cheap prepaid SIM, eSIM, or Google Voice number.

Configure access

{
  channels: {
    whatsapp: {
      dmPolicy: "allowlist",
      allowFrom: ["+15551234567"],
      groupPolicy: "allowlist",
      groupAllowFrom: ["+15551234567"]
    }
  }
}

Replace with your actual phone number (international format).

Link WhatsApp

openclaw channels login --channel whatsapp

QR code appears in terminal. On the assistant's phone, open WhatsApp ‚Üí Settings ‚Üí Linked Devices ‚Üí Link a Device ‚Üí scan the QR.

Start messaging

Text the assistant's number from your personal phone. If using pairing mode, approve:

openclaw pairing list whatsapp
openclaw pairing approve whatsapp <CODE>

Tips: If WhatsApp disconnects, re-link with openclaw channels login --channel whatsapp. Disable read receipts with sendReadReceipts: false if you don't want auto blue ticks.

6. Connecting to Discord

Create a Discord bot

Discord Developer Portal ‚Üí New Application ‚Üí Bot ‚Üí Add Bot ‚Üí copy token. Enable Message Content Intent and Server Members Intent under Privileged Gateway Intents.

Invite the bot

OAuth2 ‚Üí URL Generator ‚Üí scopes: bot + applications.commands ‚Üí permissions: View Channels, Send Messages, Read Message History, Embed Links, Attach Files. Open the generated URL and authorize on your server.

Configure and start

{
  channels: {
    discord: {
      enabled: true,
      token: "YOUR_DISCORD_BOT_TOKEN",
      dmPolicy: "pairing",
      groupPolicy: "allowlist",
      guilds: {
        "YOUR_SERVER_ID": {
          requireMention: true,
          users: ["YOUR_USER_ID"]
        }
      }
    }
  }
}

Enable Developer Mode in Discord to copy IDs (right-click ‚Üí Copy ID).

openclaw gateway restart
openclaw pairing list discord
openclaw pairing approve discord <CODE>

7. Setting Up Your First Skills

Skills are what make your assistant actually useful beyond conversation. They're instruction files that teach your AI how to use specific tools.

How skills work

Each skill is a folder with a SKILL.md file containing a name, description, and instructions. OpenClaw loads them from three places (highest priority first):

  1. Workspace skills ‚Äî ~/.openclaw/workspace/skills/
  2. Managed skills ‚Äî ~/.openclaw/skills/
  3. Bundled skills ‚Äî come with OpenClaw (web search, browser, files, etc.)

Install from ClawHub

ClawHub is the public skill registry:

clawhub install <skill-slug>    # install a skill
clawhub update --all             # update all installed

Essential setup

  • Web search ‚Äî bundled, but needs a Brave API key: openclaw configure --section web
  • Browser ‚Äî bundled. Navigate sites, fill forms, take screenshots.
  • Exec ‚Äî bundled. Run shell commands. (See Security section.)

Create your own

Make a folder in ~/.openclaw/workspace/skills/my-skill/ with a SKILL.md:

---
name: my-custom-skill
description: Does something useful
---

# My Custom Skill

Instructions for the AI on how to use this skill...

8. Security Hardening Basics

Your AI can run commands, read files, and send messages. Lock it down.

Always use allowlists

Never run open-to-the-world. Always set allowFrom:

{
  channels: {
    whatsapp: { dmPolicy: "allowlist", allowFrom: ["+15551234567"] },
    telegram: { dmPolicy: "pairing" }
  }
}

Use pairing mode

dmPolicy: "pairing" is the default and safest option. Unknown senders get a one-time code that you must manually approve. Nobody talks to your AI without your permission.

Consider sandboxing

Run tool executions in Docker containers to limit blast radius:

{
  agents: {
    defaults: {
      sandbox: { mode: "non-main", scope: "session" }
    }
  }
}

More security tips

  • Keep gateway auth enabled ‚Äî token auth is set by default.
  • Update regularly: npm install -g openclaw@latest
  • Use Tailscale for remote access (don't expose your gateway to the internet).
  • Review third-party skills before installing ‚Äî treat them like browser extensions.
  • Back up your workspace as a private git repo.

9. Common Problems and Fixes

"openclaw: command not found"

Add the global npm bin directory to your PATH:

# Add to ~/.zshrc (Mac) or ~/.bashrc (Linux):
export PATH="$(npm prefix -g)/bin:$PATH"

Then restart your terminal.

Gateway won't start

openclaw doctor         # diagnose config issues
openclaw logs --follow  # see real-time logs

Common causes: invalid config (unknown keys), port 18789 in use, missing API key.

Telegram bot not responding

  • Verify bot token in config
  • openclaw channels status to check connection
  • In groups: disable privacy mode via BotFather (/setprivacy ‚Üí Disable)
  • Check openclaw pairing list telegram ‚Äî you may need to approve yourself

WhatsApp disconnected

openclaw channels login --channel whatsapp

Scan QR code again. WhatsApp Web sessions can drop periodically.

Messages arrive but no response

  • Check openclaw logs --follow for errors
  • Verify API key is valid and has credits
  • Confirm your number/ID is in allowFrom
  • Run openclaw doctor

"sharp build errors" during install

SHARP_IGNORE_GLOBAL_LIBVIPS=1 npm install -g openclaw@latest

General debugging checklist

1.openclaw doctor ‚Äî catches most config issues
2.openclaw status ‚Äî is the gateway running?
3.openclaw channels status ‚Äî are channels connected?
4.openclaw logs --follow ‚Äî real-time log stream
5.openclaw gateway restart ‚Äî the classic fix
